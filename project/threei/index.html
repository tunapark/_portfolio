'기기 연동 UX' 개선

사용자 이탈률 감소를 위한 PM 성장기

PM 인턴으로서 하드웨어와 크롬 익스텐션 연동 과정의 높은 이탈률 문제를 발견하고, UX 리서치, 데이터 분석, 와이어프레임 설계를 통해 개선을 주도하여 사용자 이탈률 12% 감소를 달성한 프로젝트입니다.

1. 문제 정의: 높은 이탈률의 원인을 찾아서

"왜 사용자들은 연동 과정에서 포기할까?" 데이터와 CS에서 문제의 실마리를 찾았습니다.

현황 분석

자사 하드웨어와 크롬 익스텐션의 연동 과정에서 사용자 이탈이 높게 발생하는 상황을 데이터로 확인했습니다.

문제 지점 발견

사용자가 직접 앱과 하드웨어를 연동해야 하는 복잡하고 불친절한 프로세스가 핵심 문제임을 발견했습니다.

가설 수립

CS 데이터 분석 결과, "직관적인 온보딩과 UX 개선을 통해 이탈률을 15% 이상 감소시킬 수 있다"는 가설을 세웠습니다.

프로젝트 목표

기기 연동 완료율을 높이고 사용자 이탈률을 최소 15% 이상 감소시켜, 매끄러운 고객 경험을 제공하는 것을 목표로 설정했습니다.

2. 검증 절차와 기준: 성공을 어떻게 측정할 것인가

가설을 객관적으로 검증하기 위해 명확한 지표와 기준을 설정했습니다.

KPI 설정

핵심 성과 지표(KPI)로 '기기 연동 완료율', '사용자 이탈률', '관련 CS 문의 수'를 설정하여 개선 효과를 측정했습니다.

데이터 비교 분석

개선안 배포 전후의 데이터를 비교 분석하여, UX 개선이 실제 사용자 행동에 미치는 영향을 정량적으로 파악했습니다.

성공 기준 정의

'사용자 이탈률 15% 이상 감소'와 '관련 CS 문의 수의 유의미한 감소'를 프로젝트 성공의 기준으로 정의했습니다.

3. 개선 과정: 가설을 해결책으로

문제 정의와 데이터 분석을 바탕으로 구체적인 해결책을 설계하고 실행했습니다.

솔루션 실행

문제 현황 파악 및 개선점 발굴

사용자 데이터와 CS를 분석하고 경쟁사 UX를 리서치하여 서비스의 현황과 개선 포인트를 구체화했습니다.

UI/UX 개선 주도

사용자 이탈을 막기 위한 직관적인 온보딩 화면 와이어프레임을 직접 작성하고 UI/UX 개선안을 도출했습니다.

디자이너, 개발자와 협업

도출된 개선안과 와이어프레임을 바탕으로 디자이너, 개발자와 긴밀하게 협업하며 실제 개발을 진행했습니다.

핵심 개선 사항

  • 온보딩 프로세스 도입: 사용자가 연동 과정을 쉽게 이해하고 따라올 수 있도록 단계별 가이드를 제공하는 온보딩 화면을 새롭게 도입했습니다.
  • 직관적인 연동 UI/UX: 복잡했던 연동 단계를 최소화하고, 시각적으로 명확한 인터페이스를 설계하여 사용자가 직관적으로 기기를 연동할 수 있도록 개선했습니다.
  • 오류 안내 및 도움말 강화: 연동 과정에서 발생할 수 있는 문제에 대한 명확한 오류 메시지와 해결 방법을 안내하여 사용자의 불편을 최소화했습니다.
  • 명확한 CTA 버튼: 각 단계에서 사용자가 수행해야 할 행동을 명확하게 인지할 수 있도록 Call-to-Action 버튼의 문구와 디자인을 개선했습니다.

4. 성과: 데이터로 증명된 UX 개선 효과

개선안 배포 이후, 사용자 이탈률은 눈에 띄게 감소했으며 관련 CS 문의 또한 급감했습니다.

결론 및 회고

본 프로젝트는 PM으로서 데이터를 기반으로 문제를 정의하고, 사용자 중심의 해결책을 설계하며, 유관부서와 협업하여 실질적인 비즈니스 성과를 창출한 경험입니다. 특히 온보딩 도입과 직관적인 프로세스 설계가 사용자 경험 개선에 얼마나 중요한지 체감했으며, 가설 수립-실행-검증의 사이클을 통해 성장할 수 있었던 값진 기회였습니다.

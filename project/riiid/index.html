'데이터 기반 혜택 정책' 설계

부정 사용자 30% 감소시킨 데이터 전략

Data Strategy Intern으로서 로그/구매 이력 등 데이터 분석을 통해 부정 사용자를 정의하고, 정책 개선을 제안하여 혜택 비용 손실율을 크게 낮추고 부정 사용률 30% 감소를 달성한 프로젝트입니다.

1. 문제 정의 및 가설 수립

"왜 동일 고객이 여러 계정으로 혜택을 받을까?" 어뷰징의 근본 원인을 파헤쳤습니다.

문제 상황

동일 고객이 다수 계정을 생성해 혜택을 반복 수령하는 부정 사용자가 지속적으로 발생하고 있었습니다.

기존 방식의 한계

단순 블랙리스트 방식으로는 변칙적인 부정 사용자를 실시간으로 탐지하고 대응하는 데 한계가 있었습니다.

가설 수립

접속 패턴, 복귀 주기, 디바이스 정보 등 로그를 통합 분석하면 어뷰징 행동의 공통 기준을 설계할 수 있다는 가설을 세웠습니다.

프로젝트 목표

데이터 기반 부정 사용자 분류 체계를 설계하고, 이를 통해 혜택 비용 손실률을 감소시키는 것을 목표로 설정했습니다.

2. 검증 절차와 기준: 성공을 어떻게 측정할 것인가

가설을 객관적으로 검증하기 위해 명확한 지표와 분석 방법을 정의했습니다.

KPI 설정

핵심 성과 지표(KPI)로 '혜택 남용 계정 수'와 '혜택 비용 손실률'을 설정하여 개선 효과를 정량적으로 추적했습니다.

분석 방법론

로그 데이터를 기반으로 클러스터링 및 스코어링 모델을 활용하여 부정 사용 의심 계정을 정량적으로 추출했습니다.

성공 기준 정의

'부정 사용률 20% 이상 감소' 및 '혜택 비용 손실률의 유의미한 감소'를 프로젝트 성공 기준으로 정의했습니다.

3. 분석 및 정책 설계: 데이터를 해결책으로

데이터 분석을 바탕으로 부정 사용자를 분류하고, 이를 기반으로 효과적인 정책 개선안을 도출했습니다.

솔루션 실행 과정

이상 행동 패턴 분석

로그, 접속 기기, 구매 이력 등 다양한 데이터를 통합하여 다중 계정 사용, 비정상 복귀 등 이상 행동 패턴을 분석했습니다.

부정 사용자 분류 체계 설계

클러스터링과 스코어링 모델을 적용하여 부정 사용자를 위험도에 따라 분류하고 세그먼트화하는 체계를 설계했습니다.

혜택 정책 개선안 제안

분석 결과를 바탕으로 부정 사용자의 혜택 획득을 제한하고, 비용 손실을 최소화하는 새로운 혜택 정책을 설계하여 제안했습니다.

핵심 결과물

  • 이상 행동 지표 발굴: 동일 IP/기기 다수 계정 접속, 짧은 복귀 주기 등 부정 사용을 식별할 수 있는 핵심 데이터 지표를 발굴했습니다.
  • 부정 사용자 세그먼트화: 분석 모델을 통해 '상습 어뷰저', '의심 사용자', '정상 사용자'로 그룹을 나누어 차등 대응 전략의 기반을 마련했습니다.
  • 데이터 기반 혜택 정책: 사용자 세그먼트별로 혜택 지급 조건과 한도를 조정하는 새로운 정책을 제안하여 실제 서비스에 반영했습니다.
  • 모니터링 대시보드 제안: 부정 사용률과 정책 효과를 지속적으로 추적할 수 있는 모니터링 체계를 제안하여 빠른 의사결정을 지원했습니다.

4. 성과: 데이터로 증명된 정책 개선 효과

정책 개선 이후, 부정 사용 관련 지표들이 눈에 띄게 개선되었으며 실제 혜택 조건 조정에 반영되었습니다.

결론 및 회고

본 프로젝트는 데이터 분석으로 비즈니스 문제를 정의하고, 정량적 모델링으로 해결책을 도출하여 실제 정책에 반영시킨 경험입니다. 숨겨진 데이터 패턴을 발견하여 부정 사용자를 효과적으로 분류하고, 데이터 기반 의사결정이 비즈니스 손실 감소와 서비스 건전성 향상에 기여하는 과정을 체감했습니다. 분석에서 끝나는 것이 아닌, 실제 정책 변화를 이끌어냈다는 점에서 성장할 수 있었던 값진 기회였습니다.

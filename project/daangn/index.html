'키워드 알림 서비스' 개선

데이터 분석을 통한 문제 해결 사례

당근마켓 데이터 분석가 인턴십 기간 동안 '키워드 알림' 기능의 낮은 클릭률 문제를 데이터 기반으로 분석하고, A/B 테스트를 통해 2배 이상의 성과 개선을 이끌어낸 프로젝트 경험을 소개합니다.

1. 문제 정의: 데이터에서 기회 발견

"사용자들은 알림을 받고도 왜 클릭하지 않을까?" 전환율 분석으로 문제 지점을 특정했습니다.

현황 분석

단계별 전환율과 고객 행동 빈도 등 정량 데이터를 분석하여 서비스 현황을 진단했습니다.

문제 지점 발견

가장 큰 전환율 하락이 발생하는 '키워드 알림 수신 → 클릭' 퍼널을 핵심 문제 구간으로 식별했습니다.

문제 구체화

문제점을 '키워드 알림의 낮은 클릭률(CTR)로 인한 사용자 만족도 및 서비스 활성화 저하'로 명확히 정의했습니다.

프로젝트 목표

핵심 지표인 CTR을 개선하여 사용자 경험을 향상시키고, 나아가 서비스의 전반적인 가치를 증대시키는 것을 목표로 설정했습니다.

2. 원인 분석: '왜'라는 질문에 답하기

데이터와 사용자 경험, 팀 논의를 통해 문제의 근본 원인을 파고들었습니다.

정량적 분석

키워드 알림 관련 CS 데이터를 분석하여 사용자들이 느끼는 불편함의 종류와 빈도를 파악했습니다.

정성적 접근

서비스를 직접 사용(Dogfooding)하고 팀원들과 논의하며 사용자의 입장에서 문제 원인에 대한 가설을 수립했습니다.

핵심 가설 수립

"불필요한 알림이 많아 피로도가 높고, 정작 원하는 정보는 놓친다"는 핵심 가설을 설정했습니다.

3. 개선 과정: 가설을 해결책으로

분석 결과를 바탕으로 구체적인 해결책을 제안하고 검증 방안을 설계했습니다.

솔루션 도출

분석 보고서 작성 및 공유

'키워드 알림 현황 분석 보고서'를 통해 팀 전체에 문제의 심각성과 데이터 기반 근거를 공유했습니다.

개선 방향성 수립

"알림의 효용성은 올리고, 알림 피로도는 줄이자"는 명확한 방향성을 도출하여 팀의 공감대를 형성했습니다.

기능 우선순위 결정

고객 설문조사와 CS에서 가장 많이 언급된 니즈를 바탕으로 '키워드 알림 필터 기능' 개발의 우선순위를 도출했습니다.

A/B 테스트 설계

  • 실험 목표: 개선된 키워드 알림 서비스의 성과 검증
  • 핵심 지표 (Primary)
    • 유저 평균 키워드 알림 클릭률 (CTR)
  • 보조 지표 (Secondary)
    • 알림을 통한 거래 액션(채팅 등) 발생 수
  • 성공 기준
    • 실험군(B)의 핵심 지표가 통계적으로 유의미하게 대조군(A)보다 높을 경우

4. 성과: 데이터로 증명된 2배의 성장

A/B 테스트 롤아웃 이후, 개선된 키워드 알림 서비스는 명확하고 측정 가능한 성과를 보였습니다.

본 프로젝트는 데이터를 통해 주도적으로 문제를 발견하고, 논리적 가설 수립과 실험을 통해 구체적인 해결책으로 비즈니스 성과를 창출한 경험이었습니다. 데이터 분석가의 역할이 어떻게 가설 검증과 협업을 통해 서비스 개선에 직접적으로 기여할 수 있는지 깊이 배울 수 있었던 값진 기회였습니다.
